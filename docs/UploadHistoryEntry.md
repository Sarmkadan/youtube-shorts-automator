# UploadHistoryEntry

Represents a single entry in the upload history for the YouTube Shorts automator. Each entry records an attempt to upload a video file, including its local file name, the resulting YouTube video identifier (if successful), the upload timestamp, the status of the operation, and any error message. The `CreatedAt` field captures when the record was first persisted.

## API

### `public int Id`
- **Type:** `int`  
- **Purpose:** Unique identifier for this history entry. Typically assigned by the database or storage layer.  
- **Throws:** Never (auto‑property).

### `public string VideoFileName`
- **Type:** `string`  
- **Purpose:** The name (or path) of the video file that was uploaded. This value is required and should not be null.  
- **Throws:** Never (auto‑property). Setting to `null` is allowed at runtime but may cause issues downstream.

### `public string? YouTubeVideoId`
- **Type:** `string?` (nullable)  
- **Purpose:** The YouTube video identifier returned after a successful upload. `null` if the upload failed or has not yet completed.  
- **Throws:** Never (auto‑property).

### `public DateTime UploadedAt`
- **Type:** `DateTime`  
- **Purpose:** The date and time when the upload was performed (or attempted).  
- **Throws:** Never (auto‑property).

### `public UploadHistoryStatus Status`
- **Type:** `UploadHistoryStatus` (enum)  
- **Purpose:** The current status of the upload operation (e.g., `Pending`, `Success`, `Failed`).  
- **Throws:** Never (auto‑property).

### `public string? ErrorMessage`
- **Type:** `string?` (nullable)  
- **Purpose:** A human‑readable error message if the upload failed. `null` when the upload succeeded or is still pending.  
- **Throws:** Never (auto‑property).

### `public DateTime CreatedAt`
- **Type:** `DateTime`  
- **Purpose:** The date and time when this history entry was created (typically when the record was first saved).  
- **Throws:** Never (auto‑property).

## Usage

### Example 1: Creating and saving a new upload history entry

```csharp
var entry = new UploadHistoryEntry
{
    VideoFileName = "shorts/my_video.mp4",
    YouTubeVideoId = null,
    UploadedAt = DateTime.UtcNow,
    Status = UploadHistoryStatus.Pending,
    ErrorMessage = null,
    CreatedAt = DateTime.UtcNow
};

// Assuming a DbContext or repository
context.UploadHistory.Add(entry);
await context.SaveChangesAsync();
```

### Example 2: Querying failed uploads and inspecting error details

```csharp
using var context = new AppDbContext();
var failedEntries = context.UploadHistory
    .Where(e => e.Status == UploadHistoryStatus.Failed)
    .OrderByDescending(e => e.UploadedAt)
    .Take(10)
    .ToList();

foreach (var entry in failedEntries)
{
    Console.WriteLine($"File: {entry.VideoFileName}");
    Console.WriteLine($"Error: {entry.ErrorMessage ?? "No details"}");
    Console.WriteLine($"Attempted at: {entry.UploadedAt:yyyy-MM-dd HH:mm:ss}");
}
```

## Notes

- **Nullable members:** `YouTubeVideoId` and `ErrorMessage` are nullable. `YouTubeVideoId` is expected to be non‑null only when `Status` is `Success`. `ErrorMessage` is expected to be non‑null only when `Status` is `Failed`. Consumers should check the `Status` before relying on these values.
- **DateTime kind:** The `DateTime` properties (`UploadedAt`, `CreatedAt`) do not enforce a specific `DateTimeKind`. For consistency, it is recommended to store and compare values in UTC.
- **Thread safety:** This type is a plain data object with public setters. It is not inherently thread‑safe. Concurrent reads and writes to the same instance should be synchronized externally. When used as an entity in a database context, thread safety is typically handled by the data access layer.
- **Id assignment:** The `Id` property is typically generated by the persistence layer (e.g., auto‑increment). Setting it manually may cause conflicts unless the storage system supports explicit identifiers.
